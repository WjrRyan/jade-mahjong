# Jade Mahjong Technical Spec

## 1. 文档目的

本文档用于描述 Jade Mahjong 当前版本的技术实现边界、核心模块职责、关键规则落点，以及宝牌/连击系统的实现方式，便于后续维护、重构或继续迭代。

## 2. 技术栈

- React 19
- TypeScript
- Vite
- Vitest
- ESLint
- `localStorage` 持久化

## 3. 代码结构

```text
app/src/
  components/   复用 UI 组件
  data/         关卡配置、牌型元数据
  game/         核心规则引擎、视口与测试
  screens/      首页、对局页、结果页
  storage/      进度与设置持久化
  test/         测试初始化
  types/        共享类型定义
```

## 4. 模块职责

### 4.1 `data/`

负责提供静态配置：

- 牌型元数据
- 关卡定义
- 难度、步数与时间基准

### 4.2 `game/`

负责承载游戏规则：

- 可解开局生成
- 自由牌判断
- 匹配与移除
- 提示
- 洗牌
- 停局恢复
- 撤回
- 计分、宝牌、连击

原则：

- 核心规则尽量收敛在引擎层
- UI 层不重复推导业务规则

### 4.3 `screens/`

负责页面级渲染：

- 首页大厅
- 对局 HUD 与棋盘
- 通关结果页

### 4.4 `storage/`

负责将下列数据持久化到本地：

- 已解锁关卡
- 每关最佳成绩
- 设置项

## 5. 关键类型

### 5.1 `LevelConfig`

描述关卡定义：

- `id`
- `name`
- `difficulty`
- `slots`
- `tilePairs`
- `parMoves`
- `parTimeSec`

### 5.2 `GameState`

对局运行时状态，当前已包含：

- 当前关卡信息
- 全部牌实例
- 当前选中牌
- 步数、时间
- 提示与洗牌次数
- 历史快照
- 最近一次匹配
- 分数、连击、最高连击
- 当前宝牌与宝牌消除次数
- 最近一次清除时间与最近一次得分事件

### 5.3 `LevelResult`

用于通关结算与持久化的摘要结果，包含：

- `moves`
- `timeSec`
- `stars`
- `score`
- `bestCombo`
- `doraMatches`

## 6. 核心规则实现

### 6.1 可解关卡生成

实现位置：

- `app/src/game/gameEngine.ts`

实现思路：

1. 将 slot 布局映射为可操作的位置集合
2. 递归搜索合法的移除计划
3. 在多个可解方案中对开局空间分散度做有限评分
4. 将牌型按成对方式映射回已验证可解的移除计划

收益：

- 保证关卡存在解
- 控制开局观感不要过于机械

### 6.2 自由牌判定

规则：

- 上方无覆盖
- 左右至少一侧开放

实现位置：

- `isTileFree()`

### 6.3 匹配与移除

实现入口：

- `selectTile()`

处理流程：

1. 检查状态是否允许操作
2. 检查目标牌是否为自由牌
3. 若未选择第一张牌，则记录选中
4. 若已选中第一张牌，则检查牌型是否相同
5. 匹配成功时移除两张牌，增加步数，并推进得分与连击

### 6.4 提示逻辑

实现入口：

- `findAvailableMatch()`
- `revealHint()`
- `markHintUsed()`

设计要点：

- 先找到当前可执行配对
- 自动准备第一步选中状态
- 仅把剩余那张作为提示目标

### 6.5 洗牌与停局恢复

普通洗牌：

- `shuffleFreeTiles()`
- 仅影响当前自由牌

停局恢复：

- `smartShuffle()`
- 在剩余活跃牌中重新分配牌型，稳定暴露至少一组可消配对

## 7. 宝牌与连击系统

### 7.1 宝牌定义

每局开始时：

- 从当前实际出现在棋盘上的牌型集合中随机选取 1 个
- 该牌型在整局中保持固定

实现位置：

- `chooseDoraKind()`

状态字段：

- `doraKind`
- `doraMatches`

### 7.2 得分规则

当前规则：

- 普通配对基础分：`100`
- 宝牌额外奖励：`150`

如果当前配对命中宝牌，则本次基础奖励为：

- `100 + 150 = 250`

### 7.3 连击规则

判定窗口：

- `4000ms`

倍率规则：

- 1 连击：`x1.0`
- 2 连击：`x1.0`
- 3 连击：`x1.2`
- 4 连击：`x1.4`
- 5 连击及以上：`x1.6`

实现位置：

- `getComboMultiplier()`
- `selectTile()`
- `tickTimer()`
- `resetComboState()`

### 7.4 连击中断条件

以下行为会清空当前连击：

- 点错牌或未完成有效匹配
- `Hint`
- `Shuffle`
- `Smart Shuffle`
- 超过连击时间窗

### 7.5 最近一次得分事件

为了让 UI 能展示即时反馈，引擎保存 `lastScoreEvent`，内容包括：

- 基础分
- 宝牌加分
- 最终结算分
- 当前连击
- 当前倍率
- 是否命中宝牌
- 是否发生连击重置

## 8. 持久化设计

实现位置：

- `app/src/storage/progressStore.ts`

持久化内容：

- `unlockedLevel`
- `bestResultsByLevel`
- `settings`

兼容策略：

- 读取旧结构时，对新增成绩字段提供默认值
- 保持旧数据不因为结构升级而直接损坏

### 9.1 首页

应展示：

- 产品入口
- 进度统计
- 设置项

### 9.2 对局页

应展示：

- 关卡信息

- 当前连击与倍率

- 最近一次得分提示

### 9.3 结果页

应展示：

- 最终分数
- 步数
- 时间
- 星级
- 最佳连击
- 宝牌消除次数

## 10. 测试策略

现有测试重点覆盖：

- 自由牌判定
- 匹配与移除
- 开局牌型分布
- 洗牌与恢复
- 提示行为
- 撤回行为
- 20 关完整通路
- 进度存储
- 宝牌与连击相关计分

## 11. 已知维护建议

- 继续把规则集中在引擎层，不要在 screen 组件中复制规则判断
- 后续若扩展更多得分项，应先扩展 `GameState` / `LevelResult` 再推进 UI
- 若继续扩展产品系统，优先补稳定的类型与测试，再接入页面层
- 若要支持线上排行榜或云存档，可在 `storage/` 层之上抽象新的持久化接口
