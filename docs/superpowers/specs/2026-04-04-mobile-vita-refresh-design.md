# Jade Mahjong Mobile Vita Refresh Design

## 背景

当前版本的 Jade Mahjong 已经具备完整可玩的流程，但视觉表现仍然偏“功能可用型前端”，离移动端休闲游戏该有的精致感、沉浸感和奖励反馈还有明显差距。

用户目标非常明确：

- UI 更接近参考的 Vita Mahjong 移动端风格
- 但不是机械模仿，而是做到“同类气质下更精致、更吸引人”
- 以移动端体验为优先，而不是桌面网页式布局

## 设计目标

本轮视觉重构聚焦五个方向：

1. 强化首页的“游戏开场感”，让进入游戏更像进入一款成品手游
2. 强化对局页的“牌桌沉浸感”，让棋盘和卡牌有明确的层次、材质与光感
3. 提升交互控件层级，让主操作、辅助操作、状态信息更容易一眼分辨
4. 提升结果页奖励感，让通关反馈更像一段短暂庆祝，而不是普通信息页
5. 保持现有代码结构尽量稳定，不为了做视觉效果把业务逻辑搅乱

## 视觉方向

整体风格定为：

- 温暖、通透、偏玉石与金箔质感
- 更强的游戏化装饰边界
- 保留轻松亲和，而不是走厚重宫廷风
- 在移动端上强调“单手可读、按钮明确、板块有呼吸感”

关键词：

- jade
- lacquer
- warm gold
- silk glow
- floating cards
- premium casual game

## 页面级方案

### 1. 首页

首页从“信息堆叠”升级成“带仪式感的开场大堂”：

- 顶部使用更强的英雄区，加入发光背景、玉石标题氛围和引导文案
- 统计卡片做成更像游戏大厅资源卡，而不是普通 dashboard
- 设置区收敛成更轻、更像游戏面板的次级模块
- 关卡区增强层次感，突出当前可玩的下一关与已完成成绩

重点不是做复杂动画，而是通过：

- 更强的背景层
- 更明确的 CTA
- 更细的卡片层级

把首页变成“值得点击开始”的游戏入口。

### 2. 对局页

对局页是本轮重构的核心。

改造重点：

- 顶部状态栏更像游戏 HUD，而不是普通网页表头
- 棋盘区域做成真正的“桌面”而不是浅色容器
- 卡牌加入更明显的体积感、阴影、边缘高光和选中反馈
- 操作按钮区改成更适合拇指点击的底部工具带

棋盘应呈现：

- 深浅层次更丰富的木质/漆面背景
- 轻微内阴影和外围高光
- 牌面浮在棋盘之上的感觉

卡牌应呈现：

- 更细的边框与厚度感
- 被选中、被提示、可点击状态之间更明显的视觉差异
- 保持当前 SVG 牌面资源，但包装方式更高级

### 3. 结果页

结果页需要更有奖励感：

- 标题和文案更像“通关庆祝”
- 星级区域更突出
- 主按钮更明确地承接“下一关”
- 整体背景和卡片光感更明显

### 4. 全局背景与动效

整站背景不再只是浅色渐变，而是加入：

- 更明显的玉石雾面渐变
- 局部金色光晕
- 轻纹理或柔和的层叠阴影

动效策略：

- 页面入场做轻量渐显与上浮
- 卡片 hover / tap 保持克制但有反馈
- 不引入复杂 JS 动画库，优先用 CSS 达成

## 代码策略

本轮优先采用“结构小改 + 样式重写”的方式，而不是推翻页面组件。

计划改动集中在：

- `src/index.css`：主题变量、全局背景、主要布局与组件皮肤
- `src/screens/HomeScreen.tsx`：首页结构增强
- `src/screens/GameScreen.tsx`：对局页结构增强
- `src/screens/ResultScreen.tsx`：结果页结构增强
- `src/components/TileFace.tsx`：牌面包装细节
- `src/App.css`：清理无用模板样式

## 成功标准

如果本轮改造成功，应该达到以下结果：

- 首页、对局页、结果页都明显更像移动端游戏，而不是普通网页
- 与参考 Vita Mahjong 在气质上接近，但视觉细节更精致
- 牌桌与卡牌的层次感显著提升
- 移动端布局更自然，底部操作区更适合手指点击
- 不影响现有游戏流程与测试
