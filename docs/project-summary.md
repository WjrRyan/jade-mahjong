# Jade Mahjong 项目总结

## 1. 项目目标

Jade Mahjong 的目标不是复刻复杂的传统麻将规则，而是做出一款移动优先、轻量进入、局内反馈明确、适合碎片化游玩的麻将连连看 Web App。

核心目标包括：

- 手机上打开即可玩，学习成本低
- 牌面清晰、点击路径明确，对年长用户也更友好
- 每关都有稳定通关路径，不制造“无解挫败”
- 在基础消除玩法上增加适度的策略感与奖励感
- 保持前端实现简洁，便于后续继续迭代

## 2. 主要实现思路

### 2.1 产品层

产品采用“三屏闭环”结构：

- 首页：承担开场、大厅、关卡选择与设置入口
- 对局页：承载核心点击、提示、洗牌、停局恢复与即时反馈
- 结果页：承接通关奖励感、成绩回顾与下一局引导

这套结构的目标是让产品从第一屏到结算都像一款完整游戏，而不是单独的玩法 demo。

### 2.2 游戏规则层

核心规则采用经典麻将连连看约束：

- 只有自由牌可点击
- 相同牌型的两张自由牌可被消除
- 全部消除即通关
- 当前无可消配对时进入停局

为了保持规则易理解，系统能力被设计为“辅助完成游戏”而不是“替玩家自动游戏”，因此保留了：

- `Hint`
- `Shuffle`
- `Smart Recovery`
- `Undo`
- `Restart`

### 2.3 引擎层

引擎的关键思路不是随机铺牌，而是：

1. 先根据关卡 slot 布局推导一条可解移除路径
2. 再将成对牌型映射回这条路径
3. 最终得到“已知可解”的开局牌面

这保证了关卡不是视觉上看起来合理、实际上却可能无解。

### 2.4 体验层

除了基础通关逻辑，项目还对体验做了两轮重点设计：

- 优化开局牌型分布，减少相同牌过于机械地紧邻出现
- 将 `Hint` 从“信息提示”改为“下一步可执行动作”

近期又新增了更像休闲手游的局内计分能力：

- 即时得分
- 连击倍率
- 每局随机宝牌

## 3. 开发过程中遇到的问题

### 3.1 随机铺牌容易导致无解

问题：

如果只按牌型随机分配到 slot，看起来布局可能没问题，但实际游玩时很容易出现死局。

解决：

引擎改为“先解后铺”。也就是先找到合法移除路径，再映射牌型，从根上保证每关存在完成路径。

### 3.2 轻量洗牌不能稳定救回停局

问题：

只打散当前自由牌虽然简单，但对部分停局没有足够恢复能力。

解决：

将工具分成两个层级：

- `Shuffle`：保持轻量，仅处理当前自由牌
- `Smart Recovery`：在停局时更积极地重整剩余牌型，稳定暴露出新的可消配对

### 3.3 旧版 Hint 逻辑可用，但不够“好用”

问题：

旧版 `Hint` 能找到可消对，但玩家仍然要自己判断第一下该点哪张，第二下又该接哪张，帮助并不充分。

解决：

新版 `Hint` 会自动完成第一张牌的选中，把另一张牌明确标记为下一步目标，使提示真正转化为操作建议。

### 3.4 开局观感重复、缺少变化

问题：

如果同类牌总是紧挨着出现，虽然更容易消，但会让每局的开局都显得像“系统把答案摆出来了”。

解决：

在保持可解前提下，对多个可解方案做有限比较，优先选择前几步空间更分散的方案，让局面更自然。

### 3.5 只有步数和时间，奖励感不够强

问题：

基础玩法能通，但结果页只看步数和时间，局内也缺少明显的“打得好”的反馈，整体更像工具型 demo。

解决：

新增即时计分、连击倍率与宝牌机制，让玩家在每次消除时都能感受到反馈，并在结果页形成更完整的结算体验。

## 4. 解决方案总结

项目里几个关键解决方案可以概括为：

### 4.1 用可解路径替代随机铺牌

收益：

- 稳定保证可玩性
- 降低无解风险
- 为后续难度设计提供稳定基础

### 4.2 用双层恢复工具替代单一洗牌

收益：

- 平时保留轻量辅助
- 停局时有更强恢复手段
- 兼顾策略感与可完成性

### 4.3 用“动作提示”替代“信息提示”

收益：

- 玩家更容易理解下一步
- 降低关闭高亮后的可用性损失
- 让辅助系统更具操作价值

### 4.4 用计分系统提升游戏化反馈

收益：

- 每次操作都有正反馈
- 连续成功消除会产生节奏奖励
- 通关时能展示更完整的成绩结构

## 5. 产品创新：宝牌机制

宝牌机制是当前版本最明确的产品创新点之一。

### 5.1 设计出发点

传统麻将连连看常见问题是：

- 玩家只是在机械找相同图案
- 对局节奏比较平
- 每一步之间缺少优先级判断

宝牌机制的目标不是增加复杂规则负担，而是给简单玩法加上一层轻量策略：

- 每局开始时，从当前牌面实际存在的牌型中随机选出 1 个宝牌
- 消除宝牌会获得额外奖励分
- 宝牌奖励还会被连击倍率进一步放大

### 5.2 产品价值

这会带来几个明显变化：

- 玩家会对“当前应该先消哪组”产生更明确的判断
- 对局里出现“追宝牌”与“保连击”的短线决策
- 每局的策略关注点都会略有变化，提高重复可玩性

### 5.3 为什么它适合这个项目

宝牌机制的优点是：

- 不增加学习门槛太高的新规则
- 只影响奖励，不破坏原有消除规则
- 易于在 UI 中解释和展示
- 很适合移动端休闲游戏追求的“轻策略 + 强反馈”

它让 Jade Mahjong 在基础连连看框架上更像一款真正的产品，而不是只有完成通关条件的原型。

## 6. 当前成果

当前版本已经具备：

- 首页、对局页、结果页完整闭环
- 20 个手工配置关卡
- 可解关卡铺牌逻辑
- 提示、洗牌、停局恢复、撤回、重开
- 本地进度与最佳成绩保存
- 移动端与 Senior Mode 体验优化
- 即时得分、连击倍率、宝牌奖励与结果页扩展结算

## 7. 后续建议

如果继续迭代，这个项目最值得优先推进的方向包括：

- 补齐音效、震动、轻动画，增强反馈层
- 将宝牌与连击说明做成更清晰的新手引导
- 引入种子化局面，支持复盘、分享与更稳定的难度分析
- 增加每日挑战、限时目标、连击目标等轻运营玩法
- 增加更多主题皮肤与牌面素材，强化产品辨识度
