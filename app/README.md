# Jade Mahjong

一个面向移动端、强调清晰可读与轻松节奏的麻将连连看 Web App。项目最初用于 AI 方向 take-home 作业实现，当前版本已经补充了玩法优化、测试与中文交付文档。

## 项目目标

这个项目的目标不是复刻复杂的大型麻将游戏，而是先把一个“可以直接玩、规则明确、结构可扩展”的核心版本做扎实。

当前设计重点是：

- 手机上可直接游玩
- 界面清晰、操作简单
- 年长用户也能较轻松辨认牌面与状态
- 关卡有完整的通关路径，不会随便生成无解局
- 后续方便继续扩展视觉、难度和系统能力

## 最近一轮重点优化

本轮重点处理了两个直接影响体验的问题。

### 1. 开局牌型过于固定

旧版有一个很明显的问题：很多局面里，同类牌会非常整齐地紧挨在一起。虽然这样更容易立刻消掉，但也会让开局显得过于机械，变化感不足。

这次的处理方式是：

- 保留原有“先求一个可解方案”的总体架构
- 在多个可解移除方案中做有限比较
- 优先挑选开局前几步空间更分散的方案
- 更深层仍然走快速路径，避免初始化变慢

结果是：

- 相同牌不再总是成对紧贴出现
- 开局观感更自然
- 关卡依旧保持可解

### 2. `Hint` 不够好用

旧版 `Hint` 虽然能找到匹配，但玩家得到的反馈不够直接。特别是在关闭高亮设置时，提示几乎只剩下内部状态变化，操作上帮助很弱。

现在的行为是：

- 点击 `Hint` 时先找到一个当前可消配对
- 自动选中其中一张牌
- 将另一张牌明确作为下一步目标

这样玩家再点击一次就能完成本次提示对应的匹配，`Hint` 从被动信息变成了真正可执行的辅助。

## 核心规则

### 什么是自由牌

一张牌同时满足下面条件时才可点击：

- 上方没有重叠遮挡
- 左右至少一侧开放

### 如何匹配

- 两张牌都必须是自由牌
- 两张牌必须是相同牌型
- 匹配成功后，两张牌被移除

### 状态判定

- 全部移除：`won`
- 还有牌但没有可消配对：`stalled`
- 其余情况：`playing`

## 主要交互能力

### Hint

- 给出一个当前可执行的提示
- 自动准备第一步选中状态
- 保留提示次数统计

### Shuffle

- 只打散当前自由牌的牌型
- 保持轻量工具属性

### Smart Recovery

- 在停局时重新整理剩余活跃牌
- 目标是稳定暴露新的可消配对

### Undo

- 基于历史快照恢复到上一步

### Senior Mode

- 放大牌面与视觉元素
- 提升阅读与点击舒适度

## 数据结构

### 关卡配置

关卡定义位于 `src/data/levels.ts`，包含：

- `id`
- `name`
- `difficulty`
- `slots`
- `tilePairs`
- `parMoves`
- `parTimeSec`

### 游戏状态

核心状态位于 `src/types/game.ts`，包含：

- 当前关卡
- 全部牌实例
- 当前选中牌
- 步数
- 时间
- 提示与洗牌次数
- 历史快照
- 最近一次匹配结果

### 本地进度

通过 `localStorage` 保存：

- 已解锁关卡
- 每关最好成绩
- 设置项
