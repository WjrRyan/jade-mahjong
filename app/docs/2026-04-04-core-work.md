# 2026-04-04 Core Work Summary

## Goal

Address two gameplay issues in Jade Mahjong:

- opening layouts felt too fixed because many identical tiles were placed right next to each other
- the `Hint` action existed but did not produce a reliably useful player interaction

## What Was Done

### 1. Improved board opening variety

The tile assignment flow in `src/game/gameEngine.ts` was updated so that the engine no longer always takes the first solvable removal plan it finds.

Instead, the engine now:

- explores a small set of removal-plan variants
- scores early pairs to prefer more spatial separation
- limits the deeper search to a fast path after the most visible opening steps

Result:

- opening boards feel less "pre-bundled" into adjacent identical pairs
- solvability is preserved
- search cost stays bounded enough for the full level suite

### 2. Reworked hint into an actionable assist

The hint flow was changed from "show two IDs and increment the counter" to "prepare the next move for the player."

New behavior:

- hint finds one currently playable match
- one tile from that match is auto-selected
- the partner tile is highlighted as the next tap target
- hint still increments usage metrics

Result:

- hint is useful even when visual hint highlighting is disabled
- players get a clear next action instead of a passive visual suggestion

## Main Files Updated

- `src/game/gameEngine.ts`
- `src/App.tsx`
- `src/screens/GameScreen.tsx`
- `src/game/gameEngine.test.ts`
- `src/App.test.tsx`

## Simplifications Made

- centralized hint preparation inside the game engine instead of splitting logic between the engine and UI layer
- optimized only the opening portion of removal-plan scoring rather than turning the whole solver into a heavy global optimizer

## Outcome

The gameplay now feels less repetitive at the start of a level, and the hint button now directly helps the player complete a move.
