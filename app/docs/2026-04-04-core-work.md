# 2026-04-04 核心工作记录

## 目标

今天主要处理 Jade Mahjong 中两个直接影响体验的问题：

- 开局牌型过于固定，很多相同牌会紧挨着出现
- `Hint` 功能虽然存在，但实际使用时帮助不够直接

## 本次完成内容

### 1. 优化开局牌型分布

在 `src/game/gameEngine.ts` 中，原本的做法更接近“找到一个可解方案后就直接使用”。这能保证关卡可解，但也容易让某些布局长期呈现固定、明显的配对结构。

这次调整后，引擎会：

- 探索少量可解的移除方案变体
- 对开局前几步的配对做空间分散度评分
- 优先选择更不容易出现贴邻双牌的方案
- 在更深层搜索中回退到快速路径，避免性能明显变差

结果是：

- 开局看起来不再像系统把答案直接摆好
- 同类牌分布更自然
- 依然保持关卡可解
- 20 关整套流程测试仍能通过

### 2. 重做 Hint 的交互价值

之前的 `Hint` 行为更偏向“找到可消对并记录下来”，但玩家不一定能立刻感知到下一步该做什么，特别是在关闭高亮提示设置后帮助更弱。

现在的处理方式是：

- `Hint` 先找到一个当前可消的配对
- 自动选中其中一张牌
- 把另一张牌作为下一步点击目标
- 同时保留提示次数统计

这样提示就从“被动信息”变成了“下一步操作建议”。

## 本轮做过的简化

- 将提示准备逻辑集中回游戏引擎，避免 UI 层维护过多局部状态
- 只对最影响玩家第一印象的开局步骤做牌型分布优化，而不是把整个可解搜索都做成重型优化器

## 直接结果

- 开局牌面变化感更强
- 同类牌过度贴邻的情况显著减少
- `Hint` 可以真正帮助玩家完成下一步
- 回归测试对这两个问题都已经补上保护
